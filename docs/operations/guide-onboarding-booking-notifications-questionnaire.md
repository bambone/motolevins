# Анкета для гида: блок «Запись и настройки» (RentBase)

**Назначение:** администратор клиента заполняет бриф; по ответам гид преднастраивает разделы кабинета **«Запись и расписание»** и **«Настройки» → получатели и правила уведомлений**, затем сопровождает донастройку.

| Поле | Значение |
|------|----------|
| Версия анкеты | `v1` |
| Идентификатор документа (slug) | `rentbase-appointment-notifications-v1` |
| Дата заполнения | |

**Принципы:** у каждого вопроса ниже указан **machine-slug** (стабильный ключ). Новые вопросы в следующих версиях добавляются с **новыми** slug; существующие slug **не переименовываются**. Ответы можно помечать уровнем: *обязательно / желательно / не нужно* (где уместно).

**Ориентиры в коде:** группа навигации Scheduling в `app/Providers/Filament/AdminPanelProvider.php`; получатели — `NotificationDestination`; правила — `NotificationSubscription`; события — `App\NotificationCenter\NotificationEventRegistry`; каналы — `App\NotificationCenter\NotificationChannelType`.

**Интеграция в кабинет (автоприменение):** страница **«Запись и уведомления (бриф)»** в группе «Запуск сайта» (`TenantSiteSetupBookingNotificationsPage`, маршрут `filament.admin.pages.site-setup-booking-notifications`, URL `/admin/site-setup-booking-notifications`). Черновик и отметка «применено» хранятся в `tenant_settings`: ключи JSON `setup.booking_notifications_questionnaire` и строка `setup.booking_notifications_applied_at`. Пункт чеклиста мастера запуска: `setup.booking_notifications_brief` в `NotificationsSetupItemsProvider`. Логика применения: `App\TenantSiteSetup\BookingNotificationsBriefingApplier`.
