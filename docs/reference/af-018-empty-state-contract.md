# AF-018 — Контракт пустых состояний (админки)

Цель: пустой список **объясняет причину**, **не путается с ошибкой** и **подсказывает следующий шаг**. Связано с [ux-glossary.md](ux-glossary.md) и [af-017-admin-terminology-contract.md](af-017-admin-terminology-contract.md).

## Типы пустоты (минимальная классификация)

| Тип | Смысл для пользователя | Текст / поведение |
|-----|------------------------|-------------------|
| **Первичная пустота** | Раздел ещё не заполнен | Заголовок «Пока нет …», что добавить первым, при необходимости CTA «Создать …» |
| **Пусто по фильтру/поиску** | Записи есть, но не под условия | Явно добавить мысль: сбросить фильтры / изменить поиск. В Filament один общий empty state — в описание можно добавить фразу из `AdminEmptyState::hintFiltersAndSearch()` |
| **Норма без действия** | Ожидаемо пусто (логи, история за период) | Спокойный тон: «пока не было …» без тревоги |
| **Зависимость от настройки** | Нужен другой шаг (получатели, интеграция) | Указать родительский раздел по-русски |

## Структура текста

1. **Заголовок** — что отсутствует (коротко).
2. **Описание** — почему так бывает / что это значит / что делать.
3. **CTA** (если уместно) — через `emptyStateActions()` (например `CreateAction`).

## Инженерный паттерн

- Для единообразия: класс `App\Filament\Support\AdminEmptyState`.
- Иконка: `heroicon-o-*` в духе раздела.

## Различие platform / tenant

- **Кабинет клиента** — язык задач сайта и бизнеса.
- **Консоль платформы** — можно чуть суже, но всё равно с объяснением и без сырого «No records».
