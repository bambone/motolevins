## Продукт и панели

| Внутреннее | В UI |
|------------|------|
| Tenant | **Клиент** |
| Platform panel | **Консоль платформы** |
| Tenant admin | **Кабинет клиента** |
| Public site | **Сайт для посетителей** |
| Slug | **URL-идентификатор** (в пресетах и справочниках: «URL-идентификатор (пресет)», если нужно отличить от адреса страницы) |
| Tenant (в UI кабинета клиента) | **Клиент** / **ваш сайт** — не показывать слово *tenant* конечному пользователю |
| Webhook (канал уведомлений) | **Входящий webhook (HTTP)** или пояснение рядом: «запросы от внешней системы по URL» |
| Follow-up (CRM) | **Следующий контакт** (дата/время напоминания менеджеру) |
| Target (scheduling) | **Объект в расписании** (связь календаря с сущностью бронирования) |

## Аббревиатуры в подписях

| Допустимо | Как показывать |
|-----------|----------------|
| UTM | **UTM (источник)** / **UTM (канал)** / **UTM (кампания)** — не оставлять `utm_source` как единственную подпись |
| CRM | **CRM** с контекстом («заявка в CRM») или «Заявка» в кабинете |
| SEO | **SEO** + пояснение в секции при первом появлении |
| SSL | **SSL-сертификат** в таблицах и формах, если места мало — **SSL** только вместе с заголовком раздела про домен |
| JSON (сырой payload) | **Доп. данные (JSON)** или блок «Технические данные» с пояснением |

## Правила отображения

1. Внутренний числовой `id` не делать главной колонкой без нужды.
2. Raw JSON — в блоке «Технические данные» / с пояснением; критичный смысл не только в tooltip.
3. Роли и статусы — человекочитаемые строки из PHP-карт, согласованные с этим файлом.
4. Английские режимы инфраструктуры (media write/delivery) — **русская подпись + пояснение**, внутреннее значение enum можно оставить вторым рядом в списке опций или в helper.
5. Одинаковые сущности (домен, заявка, URL-идентификатор) — одни и те же термины в platform и tenant, если контекст совпадает.

## Медиа (platform, переопределения на клиенте)

Смысл для оператора: *куда пишутся файлы* и *откуда их получает сайт посетителя*.

| Внутреннее значение | Подпись в UI (ориентир) |
|---------------------|-------------------------|
| `local_only` | Только локальное хранилище сервера |
| `r2_only` | Только облачное хранилище (R2) |
| `dual` | Двойная запись: локально и в облаке |
| `local` (delivery) | Отдача с сервера сайта |
| `r2` (delivery) | Отдача из облака / CDN |
